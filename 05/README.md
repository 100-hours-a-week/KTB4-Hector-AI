# 회고
## ResNet 모델을 불러와 새로운 이미지 데이터셋을 분류하세요
강의 교안을 보면서 진행했다. 강의 교안에서 나온 데이터 말고 다른 데이터로 해보고 싶어서 mnist 데이터를 사용했다. 하지만 강의 교안에서의 데이터와 mnist 데이터의 차원이 맞지 않았다. 아마도 mnist 데이터는 흑백으로 이루어져 있어서 그런 것 같다. 이를 모델을 데이터의 차원으로 맞춰주어 해결했고 강의 교안과 마찬가지로 마지막 2개 레이어를 삭제한 뒤 전역 풀링 레이어를 추가하였다.

## 이미지 데이터셋과 사전 훈련된 VGG16 모델을 가져와 전이 학습을 수행하세요.
ResNet과 동일한 데이터 셋으로 VGG16도 시도해보았다. 모델을 불러오고 마지막 레이어를 삭제한뒤 전역 풀링 레이어를 추가하였다. 데이터 셋에서 문제가 있었는데 ResNet을 훈련시킬 때와 동일하게 진행하면 되나 했지만 에러가 발생했다. 이유는 두 모델의 구조적 차이로 AI를 이용해 데이터 셋을 보간한 뒤 해결하였다.

## 동일한 데이터셋에서 ResNet과 VGG16을 각각 학습시켜 성능을 비교하세요.
비교 결과 ResNet50: 98.92%, VGG16: 99.50%가 나왔다. 두 모델 모두 좋은 성능을 보여주었다. 아마도 단순한 mnist 이미지의 특성인 것 같다. 나름대로 분석을 해보자면 레이어 수는 ResNET이 더 많고 파라미터는 VGG16이 더 많다. 아마도 단순한 이미지가 고해상도 이미지 셋 문제에 적합한 ResNet에 불리하게 작용된 것 같다.

## 가상 데이터셋을 생성한 뒤, GridSearch와 RandomSearch 기법으로 하이퍼파라미터 튜닝을 진행하세요.
가상 데이터는 32,32,3의 가상 이미지 데이터를 만들었다. 총 1000개의 데이터이고 순서를 섞어 80%는 훈련 20%는 테스트 데이터로 만들었다. Grid Search에서는 학습률 0.001~0.1까지 10배씩 움직이며 진행하였고 배치사이즈는 16, 32, 64 이렇게 3개를 실험해 보도록 했다. 같은 데이터로 Randomsearch도 진행하였으며 모델은 둘다 동일하게 Resnet50모델을 사용하였다. 물론 랜덤 데이터여서 연관성은 없지만 random search가 약 2배 정도 빠른 성능을 보였다. 아마 learning rate와 barch size이 외에 다른 하이퍼 파라미터를 추가하면 이 격차는 더 커질 것 같다.

## 한국어 챗봇을 만드세요. 1. 문장을 넣으면 다음 단어를 생성하는 모델을 만드세요. 2. 그 모델을 반복적으로 사용하여 완전한 문장을 답하세요. 3. FastAPI로 감싸서 웹에서 사용할 수 있도록 하세요.
기본 로직은 MiniGPT 예제를 이용하여 진행했다. MiniGPT를 해봤는데 셰익스피어 텍스트를 써서 그런지 구어체가 많았고 알아듣기 힘들었다. 성능도 그다지 좋은 느낌이 아니였다. 그래서 데이터를 Wikitext로 바꿔서 진행했다. 성능을 높이고자 최대 block사이즈를 줄이고 임베딩 차원을 늘렸다. 학습 시간이 오래 걸려 steps은 반으로 줄이고 dropout 비율도 낮췄다. 나머지는 MiniGPT예제를 조금만 수정하여 진행했다. 그 후 FastAPI를 시도했는데 기존 비쥬얼 스튜디오에서는 터미널에 실행 명령만 입력하면 서버 접속이 됐던것과 달리 코랩에서는 오류가 나왔다. AI의 도움을 받아 해결하였고 과정은 post명령 -> encode -> model.generate-> decode -> Json응답순으로 진행되게하였다. 

## 느낀점 + 어려웠던 점
1. 전이 학습 시에 모델에 따라 학습 데이터가 다를 것이다. 이를 새로운 task에 다시 학습 시킬때 차원 문제가 많이 발생했다. 이럴 때 어떻게 해야할지 아직 막막하다(AI 없으면 해결하기 쉽지 않을 거 같음). 아마도 내가 모델 구조를 정확히 파악하고 있지 않아 혼자서 수정이 불가능 한 것 같다.
2. Resnet50과 VGG16을 비교하면서 모든 tesk에 복잡한 모델이 더 좋다는 관념을 깼다. task에 맞게 모델을 선정해야한다는 중요성을 느꼇다.
3. 몇 주 동안 최대한 AI를 안쓰고 코드 받아쓰기를 진행했었다. 하면서는 그냥 받아쓰기하는 느낌이 강했는데 이제와서 보니 모델의 구조를 이해하는데 도움이 됐고 완벽하겐 아니여도 어디에 어떤 코드가 들어가야하는지 알게되었다.
4. 코딩테스트 연습을 하고있는데 아직 모델 코드를 짜는 것과는 다른 느낌이 든다. 코테에서 잘 안쓰는 데코레이터나 컨텍스트 매니저를 연습해야겠다.
